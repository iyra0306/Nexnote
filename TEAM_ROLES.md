## 🤝 Collaboration Guidelines

### Communication:
- Daily standup (5-10 minutes)
- Weekly team meeting (30 minutes)
- Use WhatsApp/Discord for quick questions
- Use GitHub Issues for bug tracking

### Code Review:
- All code must be reviewed by at least one teammate
- Use pull requests for major changes
- Test before pushing to main branch

### Git Workflow:
```bash
# Before starting work
git pull origin main

# After making changes
git add .
git commit -m "Clear description of changes"
git push origin main
```

### Naming Conventions:
- **Files**: camelCase (e.g., `userController.js`)
- **Components**: PascalCase (e.g., `NoteCard.jsx`)
- **Variables**: camelCase (e.g., `userName`)
- **Constants**: UPPER_CASE (e.g., `API_URL`)
